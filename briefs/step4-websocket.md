## Скоуп Шага 4 — live-доставка
Клиент подписывается на комнату по websocket и получает **новые сообщения в реальном времени**.
Отправка остаётся по HTTP (`POST .../messages`, Шаг 3) — она же **публикует** сообщение подписчикам.
WS в v0 — **приём** (не отправка); WS-send — горизонт, не сейчас.

## Эндпоинт
- `GET /chater/rooms/{id}/ws` → upgrade до websocket.
- **Гейт участника ДО апгрейда:** не-участник → 403 (переиспользуй `requireParticipant`), только потом upgrade.
- Библиотека — **`coder/websocket`** (канон Go, из founding-брифа). Пинг/понг keepalive.

## In-process pub/sub (ядро шага)
- Хаб: `room_id → множество подписчиков`. Потокобезопасно (mutex/каналы).
- В обработчик `POST .../messages` **после успешного `InsertMessage`** добавь публикацию сообщения
  подписчикам этой комнаты. Публикация не должна ронять/блокировать HTTP-путь (неблокирующая отправка;
  медленный/отвалившийся клиент — дроп/закрытие, не стопор бродкаста).
- Аккуратный lifecycle: подписка при connect, отписка + очистка при disconnect/ctx-cancel. Без гонок и утечек горутин.

## Формат кадра
JSON. Заложи конверт под будущие типы событий: `{"type":"message","message":{<как в HTTP toMessage>}}`
(в v0 только `message`; typing/presence — потом). Единый shape с HTTP-DTO — не дублируй маппинг.

## Auth на WS
Тот же token-stub (`Authorization: Bearer <handle>`) на upgrade-запросе — для агентов/Postman ок.
Браузерный путь (нельзя слать заголовок на WS) — заметь как известное ограничение, решится с реальной identity
(subprotocol/query-token), НЕ сейчас.

## DoD Шага 4
- `GET /chater/rooms/{id}/ws` апгрейдит участника; не-участник → 403 до апгрейда.
- Сообщение, отправленное `POST`, **приходит live всем подписчикам** комнаты.
- Хаб потокобезопасен; корректный connect/disconnect, keepalive; без утечек горутин.
- **Тесты `-race`:** два подписчика получают одно опубликованное сообщение; не-участник отбит; дисконнект чистит подписку.
  (`coder/websocket` тест-клиент + httptest).
- `go build/vet/test -race` зелёные; CI зелёный (вкл. sqlc-drift).
- Проверяемо вживую: подключить WS → `POST` сообщение → увидеть кадр.
- Ветка + PR, STOP → отчёт (формат кадра, решения по хабу/неблокирующей доставке, вопросы).
